1. INTRODUCTION                                            *meow-introduction*

Meow is a dark, vibrant colorscheme for Neovim with a pink accent palette.
It ships with first-class support for Treesitter, LSP semantic tokens,
NvChad/base46, and a growing set of popular plugins.

==============================================================================
2. REQUIREMENTS                                            *meow-requirements*

  • Neovim >= 0.9.0
  • `termguicolors` enabled (`:set termguicolors`)
  • Optional: nvim-treesitter for richer syntax highlighting

3. INSTALLATION                                            *meow-installation*

lazy.nvim ~
>lua
  {
    "irfanshadikrishad/meow.nvim",
    lazy = false,
    priority = 1000,
    config = function()
      require("meow").setup()
      vim.cmd.colorscheme("meow")
    end,
  }
<

vim-plug ~
>vim
  Plug 'irfanshadikrishad/meow.nvim'
<
Then in your init.lua / init.vim:
>lua
  require("meow").setup()
  vim.cmd.colorscheme("meow")
<

4. QUICK START                                              *meow-quickstart*

Minimal setup — all options are optional:
>lua
  require("meow").setup()
  vim.cmd.colorscheme("meow")
<

With options:
>lua
  require("meow").setup({
    transparent     = true,
    italic_comments = true,
    bold_keywords   = false,
    colors = {
      bg      = "#000000",
      keyword = "#ff79c6",
    },
    integrations = {
      telescope = true,
      lualine   = true,
      cmp       = true,
    },
  })
  vim.cmd.colorscheme("meow")
<

5. CONFIGURATION                                          *meow-configuration*

Pass a table to |meow.setup()|. All keys are optional; unset keys fall back to
their defaults shown below.

>lua
  require("meow").setup({
    -- Hex color overrides. Any key from |meow-palette| is valid.
    colors = {},

    -- Use a transparent background for Normal and floating windows.
    transparent = false,

    -- Render comments in italic.
    italic_comments = true,

    -- Render keywords in bold.
    bold_keywords = false,

    -- Per-integration opt-out. Set a key to false to disable that integration.
    integrations = {
      treesitter         = true,
      lsp                = true,
      gitsigns           = true,
      telescope          = true,
      fzf_lua            = true,
      which_key          = true,
      cmp                = true,
      blink_cmp          = true,
      lualine            = true,
      neo_tree           = true,
      nvim_tree          = true,
      indent_blankline   = true,
      notify             = true,
      dashboard          = true,
      nvchad             = true,
      snacks             = true,
      noice              = true,
      illuminate         = true,
      treesitter_context = true,
      rainbow_delimiters = true,
      floaterm           = true,
    },
  })
<

------------------------------------------------------------------------------
COLOR OVERRIDES                                       *meow-color-overrides*

The `colors` table accepts any key listed in |meow-palette|. Values must be
hex strings. Overrides are applied on top of the defaults before highlights
are set, so every highlight group that references the key is updated
automatically.

Example — change the background and accent:
>lua
  require("meow").setup({
    colors = {
      bg      = "#0d0d0d",
      keyword = "#c678dd",
    },
  })
<

6. API                                                            *meow-api*

------------------------------------------------------------------------------
                                                               *meow.setup()*
meow.setup([opts])

  Apply the colorscheme. Call this before `vim.cmd.colorscheme("meow")`.
  Calling `setup()` multiple times is safe — each call replaces the stored
  config and re-applies all highlights.

  Parameters:
    {opts}  (optional) |MeowConfig| table — see |meow-configuration|.

------------------------------------------------------------------------------
                                                           *meow.reload()*
meow.reload()

  Re-apply the colorscheme using the config most recently passed to
  |meow.setup()|. This is what `colors/meow.lua` calls so that
  `:colorscheme meow` always preserves your setup options.

  You generally do not need to call this directly.

------------------------------------------------------------------------------
                                                          *meow.get_colors()*
meow.get_colors()

  Return the currently active |MeowPalette| table.

  If called before |meow.setup()|, returns the palette resolved with default
  options so it is always safe to call.

  This is the recommended way to build a lualine theme or any custom highlight
  group that should stay consistent with the active meow palette:
>lua
  local c = require("meow").get_colors()
  vim.api.nvim_set_hl(0, "MyStatusline", { fg = c.fg, bg = c.bg })
<

------------------------------------------------------------------------------
                                                            *meow.extend()*
meow.extend(name, mod)

  Register a custom integration that runs on every |meow.setup()| call.

  Parameters:
    {name}  (string) Unique identifier. This key also appears in the
            `integrations` config table so users can disable it with
            `integrations = { [name] = false }`.
    {mod}   (table) Must contain a `setup(colors, config)` function.

  Example:
>lua
  require("meow").extend("my_statusline", {
    setup = function(colors, config)
      vim.api.nvim_set_hl(0, "MyStatuslineAccent", {
        fg = colors.bg,
        bg = colors.keyword,
        bold = true,
      })
    end,
  })
<

  Call `extend()` before the first `setup()` call (e.g. in the same `config`
  function) so the integration is included from the start.

>lua
  require("meow").setup({
    integrations = {
      dashboard = false,
    },
  })
<

Lualine theme ~

The lualine integration exposes a `theme()` helper in addition to applying
highlight groups:
>lua
  require("lualine").setup({
    options = {
      theme = require("meow.integrations.lualine").theme(),
    },
  })
<

8. CUSTOM INTEGRATIONS                           *meow-custom-integrations*

Any plugin can register its own integration module so users get consistent
theming without writing boilerplate:

>lua
  -- In your plugin's setup / config:
  require("meow").extend("myplugin", {
    setup = function(colors, config)
      vim.api.nvim_set_hl(0, "MyPluginBorder", {
        fg = colors.popup_border,
        bg = colors.popup_bg,
      })
      vim.api.nvim_set_hl(0, "MyPluginTitle", {
        fg = colors.keyword,
        bold = true,
      })
    end,
  })
<

Users can then disable it the same way as a built-in:
>lua
  require("meow").setup({
    integrations = { myplugin = false },
  })
<

10. NVCHAD COMPATIBILITY                                      *meow-nvchad*

Meow ships a `lua/themes/meow.lua` base46 theme definition so that NvChad's
base46 system can load it natively when `base46.theme = "meow"` is set in
your `chadrc.lua`:
>lua
  M.base46 = { theme = "meow" }
<

NvChad compat notes ~

• base46 compiles its own highlight cache and applies it in two passes: once
  at startup via `load_all_highlights()`, and again lazily when each
  integration plugin first opens (telescope, nvim-tree, tabufline, etc.).
  Meow's `colors/meow.lua` uses a double `vim.schedule` to land after base46's
  startup pass, and each integration that is affected by lazy loading registers
  a `FileType` or `BufAdd` autocmd to reapply its groups at the right moment.

• The `nvchad` integration covers: NvChad statusline (`St_*` groups for all
  modes, minimal + default + vscode styles), tabufline (`Tb*`), and nvdash
  (`NvDash*`). It does NOT require you to disable NvChad's own highlights —
  meow's autocmds win the race after base46's lazy cache loads.

• Telescope titles — NvChad's base46 defaults to a hot-pink `PromptTitle` and
  lime-green `PreviewTitle`. Meow overrides these with palette-consistent
  colors (keyword pink, function_name teal) via a `FileType TelescopePrompt`
  autocmd so base46's lazy cache never wins.

• If you use the `minimal` statusline style with `block` separators, all
  `St_<mode>_sep / _bg / _txt` groups are defined and tested.
